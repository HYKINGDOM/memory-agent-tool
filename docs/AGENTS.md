# docs

## Scope

- 这里存放项目说明、真实客户端边界、项目交付文档和使用接入文档。

## Rules

- 文档按职责分层：
  - `project/` 写项目级交付与边界
  - `usage/` 写使用方法
  - `integrations/` 写接入与真实能力说明
- 文档要和当前可运行命令保持一致，命令变更后要同步更新。
- 这里写长期知识和边界说明，不把方法论塞回根 `AGENTS.md`。
- 引用仓库文件时，优先使用当前仓库内已存在的真实路径和命令。

## Do not

- 不要在文档里宣称未验证的客户端能力。
- 不要把临时调试路径、一次性实验结论或会话过程写成正式文档。
- 不要让 README、usage、integration 三份文档彼此冲突。
